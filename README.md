# San Jose Collision ETL

## Overview
This project is an automated ETL pipeline that extracts vehicle collision location data from the San Jose public API, processes and cleans the raw dataset, and loads the structured output into a database for storage and analysis.

The pipeline is designed to transform publicly available traffic incident data into a consistent, query-friendly format that can be used for further analysis, reporting, or integration with downstream applications.

## Key Stages of the Pipeline

- **Extraction**: Collision data is pulled from the San Jose public API  
- **Transformation**: Raw records are cleaned and standardized  
- **Loading**: Processed data is stored in a database for persistent use  

## Summary
This project demonstrates practical data engineering concepts including API integration, data cleaning, pipeline design, and automated database loading.
